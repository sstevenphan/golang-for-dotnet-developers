Mốc hoàn thành

SQL-first

Trong Go, biết SQL tốt thường quan trọng hơn biết một ORM kỳ diệu

Bạn vẫn có thể dùng ORM hoặc query builder, nhưng nền tảng của hệ sinh thái Go là `database/sql`. Nó cho bạn control rõ ràng hơn về query, connection pool, transaction và mapping dữ liệu.

Chuyển mental model: Từ “domain model dẫn dắt query” sang “query dẫn dắt shape dữ liệu mà use case cần”.
Query

Đọc dữ liệu bằng query explicit và scan vào struct nhỏ

row := db.QueryRowContext(ctx,
    `select id, name, email from users where id = $1`,
    id,
)

var user User
if err := row.Scan(&user.ID, &user.Name, &user.Email); err != nil {
    return User{}, err
}

Nếu so với EF Core, phần này ít “developer comfort” hơn nhưng lại rất minh bạch: bạn biết chính xác query gì chạy, cột nào được đọc, object nào được map, và chi phí truy cập nằm ở đâu.

Transactions

Transaction nên gắn với use case, không gắn mù quáng với từng repository call

tx, err := db.BeginTx(ctx, nil)
if err != nil {
    return err
}
defer tx.Rollback()

if err := ordersRepo.Insert(ctx, tx, order); err != nil {
    return err
}

if err := auditRepo.Insert(ctx, tx, auditEvent); err != nil {
    return err
}

return tx.Commit()
  • Transaction là concern của application flow hoặc use case, không chỉ của database helper.
  • Đừng mở transaction quá sớm rồi giữ quá lâu qua I/O ngoài database.
  • Nếu cần atomicity, hãy làm cho boundary đó thật lộ rõ trong code.
