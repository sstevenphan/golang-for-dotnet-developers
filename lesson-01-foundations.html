Go mindset cho .NET developer

Trước khi học cú pháp, bạn cần hiểu triết lý của Go. Đây là phần quan trọng nhất để tránh mang nguyên cả tư duy C# sang và rồi thấy Go “thiếu tính năng”.

Mốc hoàn thành

Core Idea

Go ưu tiên độ đơn giản vận hành hơn độ giàu biểu đạt

C# là ngôn ngữ rất “feature-rich”: generics mạnh, LINQ, attribute, reflection, async/await, inheritance, pattern matching, source generators, ecosystem cực dày. Go đi theo hướng gần như ngược lại: số lượng concept ít hơn, cú pháp ngắn hơn, compile nhanh, binary gọn để deploy, và codebase dễ đọc khi đội ngũ lớn.

Tư duy quan trọng: Trong Go, “ít khả năng diễn đạt hơn” không phải khiếm khuyết. Nó là cách ngôn ngữ ép team đi về những pattern dễ đọc, dễ maintain, dễ onboard.
Mental Shift

Bạn sẽ thấy Go khác C# ở đâu nhiều nhất

C# / .NET Go Ý nghĩa thực tế
Class-centric, OOP mạnh Struct + function + interface nhỏ Thiết kế theo composition, ít hierarchy hơn.
Exception là cơ chế lỗi chính Return `error` tường minh Luồng lỗi hiện rõ ngay trong hàm gọi.
`async/await` Goroutine + channel + context Concurrency là primitive nền tảng, không chỉ là syntax sugar.
Interface thường được thiết kế trước Interface thường nhỏ và xuất hiện phía consumer Giảm coupling sớm, tránh over-abstraction.
Nhiều pattern enterprise sẵn có Chuộng sự trực tiếp, rõ ràng Nhiều chỗ không cần factory/service abstraction dày đặc.
Use Cases

Khi nào Go thường là lựa chọn rất đáng giá

Backend service hiệu năng ổn định

API service, background worker, queue consumer, gateway, data pipeline, internal tooling.

Cloud-native deployment

Build ra một binary, container gọn, startup nhanh, vận hành đơn giản trong môi trường microservice.

Hệ thống cần concurrency tự nhiên

Streaming, fan-out/fan-in, polling nhiều nguồn, I/O nặng nhưng logic không quá “enterprise framework-heavy”.

Team cần code dễ đọc

Go rất mạnh ở tính nhất quán. Nhiều codebase Go nhìn khá giống nhau, nên onboarding nhanh.

Unlearn

5 thói quen từ .NET nên bỏ sớm khi học Go

  • Đừng vội tạo quá nhiều interface. Trong Go, interface nhỏ và xuất hiện khi có nhu cầu thật.
  • Đừng cố dựng class hierarchy. `struct` + embedding + composition gần như luôn đủ.
  • Đừng chờ exception để quản lý lỗi. Hãy học cách kiểm tra `if err != nil` một cách bình tĩnh.
  • Đừng thiết kế mọi thứ theo generic abstraction từ đầu. Go chuộng code thẳng, cụ thể, rõ ý định.
  • Đừng xem verbosity của Go là lùi bước. Nó thường là chi phí nhỏ để đổi lấy tính minh bạch.
First Impression

Nếu phải tóm tắt Go cho một senior C# developer

Hãy nghĩ Go như một ngôn ngữ backend tối giản, được thiết kế để đội ngũ ship software đáng tin cậy nhanh hơn, chứ không phải để biểu diễn mọi mô hình trừu tượng tinh vi. Nó không cố thay C# ở mọi nơi. Nhưng ở services, tooling, cloud infra, data movement và concurrency thực dụng, Go cực kỳ mạnh.

Bài tập tư duy trước khi qua Bài 2 Hãy thử trả lời: nếu bỏ inheritance, exception-heavy flow và DI graph phức tạp, bạn sẽ tổ chức một service nhỏ theo cách nào? Câu hỏi này sẽ theo bạn xuyên suốt track.
