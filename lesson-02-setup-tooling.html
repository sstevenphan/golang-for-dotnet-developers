Installation

Bắt đầu bằng bản cài chính thức và editor quen tay

Hãy cài Go bản stable mới nhất từ trang chính thức, sau đó dùng VS Code với extension Go hoặc GoLand nếu bạn thích trải nghiệm IDE sâu hơn. Nếu xuất phát từ Rider / Visual Studio, bạn sẽ thấy tooling của Go nhẹ hơn, ít wizard hơn, nhưng tốc độ phản hồi rất tốt.

Khuyến nghị editor VS Code + Go extension là đủ mạnh cho phần lớn workflow: run, debug, test, format, rename, navigation.
Tư duy setup Go ít phụ thuộc IDE-specific magic hơn .NET. CLI chính là “source of truth”.
Điểm cộng lớn Cài xong thường là dùng được ngay, không phải bootstrap runtime phức tạp.
Modules

`go mod` là tương đương gần nhất với project file + NuGet dependency graph

Trong .NET bạn có `.csproj`, `PackageReference`, `dotnet restore`. Trong Go, phần này nằm ở `go.mod` và `go.sum`. File `go.mod` định nghĩa module name, version Go và dependency trực tiếp. `go.sum` giữ checksum để đảm bảo tính nhất quán khi restore package.

.NET Go Ghi chú
.csproj go.mod Định danh project/module và dependency chính.
NuGet lock / assets go.sum Lưu checksum dependency đã resolve.
`dotnet restore` `go mod tidy` / auto fetch Go thường tự fetch khi build, nhưng `tidy` rất quan trọng để dọn dependency.
First Project

Tạo ứng dụng console đầu tiên

go mod init github.com/yourname/go-for-dotnet
touch main.go
go run .

package main

import "fmt"

func main() {
    fmt.Println("Hello from Go")
}

Nếu so với .NET, bạn có thể hình dung `package main` giống như entry assembly, còn `func main()` là entry point. Không có class `Program` bắt buộc. Go bỏ hết ceremony không cần thiết ở bước này.

Command Map

Những lệnh bạn sẽ dùng hằng ngày

go run .          // chạy ứng dụng
go build .        // build binary
go test ./...     // chạy toàn bộ test
go fmt ./...      // format code
go mod tidy       // dọn dependency và sync go.mod/go.sum
go get package    // thêm hoặc nâng dependency

`go fmt` gần như là bắt buộc

Go culture rất coi trọng formatting thống nhất. Tranh luận style ít hơn rất nhiều so với nhiều ecosystem khác.

`go test ./...` là lệnh quen thuộc

Hãy nghĩ nó như cách quét toàn solution, nhưng theo package tree thay vì project graph kiểu .NET.

Takeaway

Điểm cần nhớ sau bài này

  • Go CLI là trung tâm workflow, IDE chỉ là lớp hỗ trợ.
  • `go.mod` và `go.sum` là phần bạn cần hiểu thật sớm.
  • Project Go đầu tiên rất ít ceremony, điều này là có chủ ý.
  • Hãy xem `go fmt` và `go mod tidy` như kỷ luật nền tảng của team.
