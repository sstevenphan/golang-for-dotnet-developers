Mốc hoàn thành

JSON

Go làm JSON mapping qua `encoding/json` và struct tag

type CreateUserRequest struct {
    Name  string `json:"name"`
    Email string `json:"email"`
}

Nếu bạn quen DTO trong ASP.NET Core, đây là ý tưởng tương tự nhưng tối giản hơn. Không có data annotation giàu tính năng mặc định như trong .NET; thay vào đó bạn thường tách rõ parse input, validate và business rules.

Binding

Parse request body một cách explicit và kiểm soát lỗi ngay tại handler

var req CreateUserRequest
if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
    http.Error(w, "invalid json body", http.StatusBadRequest)
    return
}

Trong Go, parse và validate thường hiện ngay trong handler hoặc một lớp mỏng gần handler. Bạn sẽ ít thấy model binding magic hơn ASP.NET Core, nhưng đổi lại luồng dữ liệu cực kỳ rõ ràng.

Configuration

Tư duy config trong Go thường đi theo env-first và constructor rõ ràng

.NET Go Ý nghĩa
`appsettings.json` + Options Environment variables + config struct Gọn hơn cho container và deploy cloud-native.
Configuration providers phong phú Thường giữ config pipeline rất mỏng Ít framework magic, nhiều explicit setup. 
type Config struct {
    HTTPPort string
    DBConn   string
    LogLevel string
}

func LoadConfig() Config {
    return Config{
        HTTPPort: getEnv("HTTP_PORT", "8080"),
        DBConn:   getEnv("DB_CONN", ""),
        LogLevel: getEnv("LOG_LEVEL", "info"),
    }
}
Logging

Đừng log theo kiểu chuỗi tự do quá sớm, hãy nghĩ theo event có key-value

logger.Info("user created",
    "user_id", user.ID,
    "email", user.Email,
    "source", "http",
)
  • Structured logging giúp filter, search và correlate dễ hơn khi chạy thật.
  • Log nên gắn business context vừa đủ, không spam toàn bộ object.
  • Phân biệt rõ log cho operational event và log cho error path.
Rule of thumb Hãy log để người trực ca production hiểu chuyện gì xảy ra sau 20 giây đọc log, không phải để dev local “cảm thấy yên tâm”.
