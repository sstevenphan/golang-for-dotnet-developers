Mốc hoàn thành

Structs

`struct` là viên gạch chính để model dữ liệu

type User struct {
    ID    int
    Name  string
    Email string
}

func (u User) DisplayName() string {
    return fmt.Sprintf("%d - %s", u.ID, u.Name)
}

Bạn có method gắn với type, nhưng không có class theo kiểu C#. Điều này khiến model trong Go “phẳng” hơn, ít ceremony hơn và thường dễ đọc hơn ở codebase service-oriented.

Receivers

Hiểu value receiver và pointer receiver là bắt buộc

type Counter struct {
    Value int
}

func (c *Counter) Increment() {
    c.Value++
}
  • Value receiver phù hợp khi dữ liệu nhỏ, immutable hoặc bạn cố ý không sửa state.
  • Pointer receiver dùng khi cần mutate state hoặc tránh copy dữ liệu lớn.
  • Trong thực tế service/backend, pointer receiver xuất hiện khá thường xuyên.
Interfaces

Go interface được implement ngầm định, không cần khai báo trước

type EmailSender interface {
    Send(to string, subject string, body string) error
}

type SmtpSender struct {}

func (s SmtpSender) Send(to string, subject string, body string) error {
    return nil
}

`SmtpSender` không cần viết thêm câu nào để “declare implements EmailSender”. Nếu method set khớp, nó tự thoả interface. Đây là khác biệt rất lớn với C#, và nó giúp decouple theo hướng nhẹ hơn.

Rule of thumb Trong Go, thường để consumer định nghĩa interface mà nó cần, thay vì provider publish interface rất sớm.
Composition

Hãy nghĩ theo composition trước, inheritance gần như không còn là công cụ chính

Bài toán C# thường làm Go thường làm
Chia sẻ hành vi Base class / abstract class Embedding, helper function, interface nhỏ
Tách dependency Interface + DI container Constructor đơn giản + interface phía consumer
Domain model Rich entities nhiều method Struct rõ ràng, service logic explicit hơn

Go không cấm abstraction, nhưng nó không thưởng cho abstraction sớm. Bạn sẽ có code tốt hơn khi bắt đầu từ concrete type, rồi mới rút interface ở nơi thật sự cần mock, swap implementation hoặc giảm coupling.

Takeaway

Những câu nên tự nhắc khi thiết kế bằng Go

  • Tôi có thật sự cần interface chưa, hay concrete type hiện tại đã đủ?
  • Tôi đang cố tái tạo inheritance vì thói quen, hay đây là nhu cầu chính đáng?
  • API này có đơn giản và đọc một lần là hiểu không?
  • Receiver của tôi nên là value hay pointer?
