Middleware

Middleware là nơi gom concern cắt ngang, nhưng đừng nhồi business logic vào đó

func Logging(next http.Handler, logger *slog.Logger) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        next.ServeHTTP(w, r)
        logger.Info("request completed",
            "method", r.Method,
            "path", r.URL.Path,
            "duration_ms", time.Since(start).Milliseconds(),
        )
    })
}

Hãy dùng middleware cho logging, tracing, auth gate, request ID, recovery và rate limiting. Còn business rule nên ở handler hoặc service layer, nếu không pipeline sẽ sớm khó đọc.

Authentication

Auth trong Go thường explicit hơn, ít “được framework lo sẵn” hơn ASP.NET Core

  • JWT, API key hoặc session đều có thể làm tốt, nhưng wiring sẽ rõ bằng code thay vì config-driven nhiều lớp.
  • Phân biệt rõ authentication và authorization, đừng gộp chung trong một middleware khổng lồ.
  • Khi auth phức tạp, hãy tách parser token, principal builder và policy check thành phần riêng.
Observability

Log thôi chưa đủ, production cần cả metrics và trace

Loại tín hiệu Dùng để làm gì Ví dụ
Logs Xem chi tiết event cụ thể Error context, payload summary, request lifecycle
Metrics Quan sát xu hướng hệ thống RPS, latency, error rate, queue depth
Traces Theo dõi request xuyên nhiều service Latency breakdown, dependency bottleneck
Nguyên tắc thực chiến Nếu service gọi database, cache, message broker hoặc downstream HTTP khác, trace sẽ giúp bạn tìm bottleneck nhanh hơn log rất nhiều.
Shutdown

Graceful shutdown là phần rất nên làm chuẩn từ sớm

ctx, stop := signal.NotifyContext(context.Background(), os.Interrupt, syscall.SIGTERM)
defer stop()

go func() {
    <-ctx.Done()
    shutdownCtx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
    defer cancel()
    _ = server.Shutdown(shutdownCtx)
}()

Trong môi trường container hoặc Kubernetes, đây không phải “nice to have”. Nó là phần giúp request đang chạy, connection đang mở và background job có cơ hội kết thúc đúng cách trước khi process chết.
